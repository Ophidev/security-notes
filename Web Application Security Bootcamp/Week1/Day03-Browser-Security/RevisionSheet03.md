# 1. Same-Origin Policy (SOP)

## Interview Definition

> Same-Origin Policy (SOP) is a browser security policy that prevents JavaScript running on one origin from reading responses from another origin.

---

## Purpose

Prevent cross-origin data theft.

---

## What is an Origin?

Origin consists of

- Protocol
- Host
- Port

All three must match.

Example

```
https://fitflow.com
```

Origin

```
Protocol : https

Host : fitflow.com

Port : 443
```

---

## Interview Questions

### Does SOP block requests?

❌ No.

It only blocks JavaScript from reading responses.

---

### Does SOP protect servers?

❌ No.

It protects browsers.

---

### Can HTML Forms bypass SOP?

✅ Yes.

Forms can submit cross-origin requests.

---

## Remember

```
Request

✔ Allowed

↓

Response

✔ Received

↓

Read Response

❌ Blocked
```

# 2. CSRF

## Interview Definition

> Cross-Site Request Forgery (CSRF) is an attack where an attacker tricks a victim's browser into sending an authenticated request to another website.

---

## Purpose

Perform unauthorized actions.

Examples

- Transfer Money
- Delete Account
- Change Email

---

## Why Does CSRF Work?

Because browsers automatically attach cookies.

---

## Browser Flow

```
Victim Logs In

↓

Session Cookie Stored

↓

Victim Visits evil.com

↓

evil.com Creates Request

↓

Browser Attaches Cookie

↓

Bank Executes Action
```

---

## Interview Questions

### Does CSRF steal cookies?

❌ No.

Browser automatically sends them.

---

### Does HttpOnly stop CSRF?

❌ No.

---

### Does SOP stop CSRF?

❌ No.

---

### Best Protection?

✅ CSRF Token

+

✅ SameSite Cookies

---

# 4. CSRF Token

## Interview Definition

> A CSRF token is a random secret generated by the server and verified on every state-changing request.

---

## Traditional HTML

```
Hidden Input
```

---

## SPA (React)

Usually

```
X-CSRF-Token Header
```

---

## Why Can't Attacker Send It?

Because SOP prevents evil.com from reading bank.com's page.

---

# 5. XSS vs CSRF

## CSRF

```
Attacker Controls

evil.com

↓

Tricks Browser

↓

Action Performed
```

---

## XSS

```
Attacker Injects JS

↓

bank.com Serves It

↓

Browser Executes It
```

# 6. CORS

## Interview Definition

> CORS is a browser security mechanism that allows a server to specify which origins are allowed to read cross-origin responses.

---

## Purpose

Allow trusted cross-origin communication.

---

## Browser Flow

```
Request

✔ Sent

↓

Response

✔ Received

↓

Browser Checks CORS

↓

Expose Response?

YES / NO
```

---

## Origin Header

Created By

```
Browser
```

Sent In

```
Request
```

---

## Access-Control-Allow-Origin

Created By

```
Server
```

Sent In

```
Response
```

---

## Interview Questions

### Does CORS block requests?

❌ No.

---

### Does CORS protect servers?

❌ No.

---

### Can JavaScript change Origin?

❌ No.

---

### Why does Postman work?

Because Postman is not a browser.

No CORS.

---

### Why does Browser fail?

Browser enforces CORS.

---

# 9. CSP

## Interview Definition

> Content Security Policy (CSP) is a browser security mechanism that allows a server to define which resources are allowed to load and execute.

---

## Purpose

Reduce XSS impact.

---

## Header

```
Content-Security-Policy
```

---

## 'self'

Means

```
Same Origin Only
```

---

## Browser Flow

```
HTML

↓

Browser Parses HTML

↓

Script Found

↓

CSP Check

↓

Execute?

YES / NO
```

---

## CSP Stops

❌ Inline Scripts

❌ Unauthorized External Scripts

---

## CSP Does NOT

❌ Remove XSS

It reduces its impact.

---

## Nonce

A random one-time value allowing trusted inline scripts to execute.

---

# 10. HttpOnly

## Interview Definition

> HttpOnly prevents JavaScript from accessing cookies using document.cookie.

---

## Protects Against

Cookie theft through XSS.

---

## Does NOT Protect Against

❌ CSRF

---

# 11. Browser Security Cheat Sheet

| Feature | Protects Against |
|----------|------------------|
| SOP | Reading Cross-Origin Data |
| CORS | Controlled Cross-Origin Reads |
| CSRF Token | Forged Requests |
| SameSite | Cross-Site Cookie Sending |
| HttpOnly | Cookie Theft |

# 12. Most Asked Interview Questions

### Q. What is SOP?

Browser policy preventing JavaScript from reading cross-origin responses.

---

### Q. What is CORS?

Server tells browser which origins may read responses.

---

### Q. Difference between SOP and CORS?

SOP blocks reading cross-origin responses.

CORS is a controlled exception.

---

### Q. Why is CSRF possible?

Browser automatically attaches cookies.

---

### Q. Difference between XSS and CSRF?

XSS injects JavaScript.

CSRF forges authenticated requests.

---

### Q. Why does Postman work but Browser doesn't?

Browser enforces CORS.

Postman doesn't.

---

### Q. What is CSP?

Browser security mechanism controlling which resources may execute.

---

### Q. Does HttpOnly stop XSS?

❌ No.

It only prevents JavaScript from reading cookies.

---

### Q. Best protection against XSS?

- Output Encoding
- Input Validation
- CSP
- HttpOnly (limits impact)
